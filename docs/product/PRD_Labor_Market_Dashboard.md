## 1. Executive Summary

**Labor Market Dashboard Calculator** — це інтерактивний веб-інструмент для моделювання сценаріїв «що якщо» на ринку праці України. Продукт дозволяє візуалізувати розподіл зайнятості за статтю, галузями та підкатегоріями у вигляді небінарного дерева з повзунками та pie-чартами, які оновлюються в реальному часі.

**Ключові характеристики:**

- **Деревоподібна структура:** 3 рівні глибини — стать → галузь → підкатегорія
- **Реальні дані:** дефолтні значення з офіційної статистики України (2021–2024)
- **Інтерактивність:** повзунки з двома режимами (автобаланс / вільний) та pie-чарти
- **Технології:** React + TypeScript, GitHub Pages, SPA

## 2. Problem Statement & Goals

### Проблема

Ринок праці України після 2022 року зазнав радикальних змін: мобілізація, міграція, руйнування промисловості та фемінізація зайнятості. Наразі немає зручного інструменту для швидкого моделювання впливу цих факторів на розподіл робочої сили по галузях.

### Цілі продукту

1. Дати можливість швидко оцінювати розподіл ринку праці за статтю та галузями
2. Моделювати сценарії «що якщо»: демобілізація, повернення біженців, зростання IT-сектору
3. Візуалізувати зміни через pie-чарти та абсолютні числа в реальному часі
4. Забезпечити простий деплой та безкоштовний хостинг (GitHub Pages)

### Success Metrics

| Метрика | Критерій |
|---|---|
| Usability | Користувач може створити сценарій за < 30 секунд |
| Performance | Всі повзунки та графіки оновлюються < 100ms |
| Accessibility | Lighthouse Accessibility score > 90 |

## 5. Data Model: Tree Structure

### TypeScript інтерфейс

```typescript
interface TreeNode {
  id: string;
  label: string;              // Назва категорії
  percentage: number;          // 0–100, від батьківського вузла
  absoluteValue: number;       // Авто-розрахунок
  genderSplit: {
    male: number;              // Відсоток чоловіків
    female: number;            // Відсоток жінок
  };
  children: TreeNode[];
  defaultPercentage: number;   // Значення за замовчуванням
  isLocked: boolean;           // Фіксація при автобалансі
}

interface DashboardState {
  totalPopulation: number;     // Загальна кількість зайнятих
  balanceMode: 'auto' | 'free';
  tree: TreeNode;
}
```

### Логіка повзунків

**Режим «Автобаланс»:** при зміні однієї категорії решта пропорційно перерозподіляються, щоб сума = 100%. Користувач може «залокувати» окремі категорії (isLocked), щоб вони не змінювалися.

**Режим «Вільний»:** кожен повзунок незалежний. Сума може не дорівнювати 100% — відображається попередження з різницею.

**Перемикач:** Toggle між режимами у верхній частині панелі керування.

## 7. Functional Requirements

### FR-01: Введення загального населення

- Числове поле для введення загальної кількості зайнятих (дефолт: 13 500 000)
- Форматування числа з розділювачами тисяч (пробіл)
- Кнопка «Скинути до дефолту»

### FR-02: Гендерний розподіл (Level 1)

- Повзунок Чоловіки/Жінки (0–100%)
- Дефолт: 52/48
- При зміні — всі дочірні вузли перераховують абсолютні числа
- Pie-chart оновлюється миттєво

### FR-03: Галузеві повзунки (Level 2)

- По одному повзунку на кожну галузь для кожної статі
- Окремі панелі «Чоловіки» і «Жінки» з власними pie-chart
- Автобаланс або вільний режим (за перемикачем)
- Можливість «залокувати» окрему галузь (іконка замка)

### FR-04: Підкатегорії (Level 3)

- Розгортання галузі по кліку — показує підкатегорії з власними повзунками
- Аналогічна логіка автобалансу
- Власний mini pie-chart для кожної розгорнутої галузі

### FR-05: Візуалізація (Pie Charts)

- Основний pie-chart: гендерний розподіл
- По 1 pie-chart для кожної статі: розподіл по галузях
- Mini pie-chart при розгортанні галузі
- Tooltips з абсолютними числами та відсотками
- Анімовані переходи при зміні значень

### FR-06: Кнопка «Скинути до дефолту»

- Повертає всі повзунки до офіційних даних України
- Модальне підтвердження перед скиданням

### FR-07: Відображення абсолютних значень

- Поряд з кожним повзунком — підпис з абсолютним числом (напр., «2 430 тис.»)
- Автоматичний перерахунок при зміні батьківського вузла

### Алгоритм автобалансу (pseudo-code)

```
function autoBalance(siblings, changedId, newValue):
  locked = siblings.filter(s => s.isLocked && s.id !== changedId)
  lockedSum = sum(locked.percentage)
  remaining = 100 - newValue - lockedSum
  
  unlocked = siblings.filter(s => !s.isLocked && s.id !== changedId)
  oldUnlockedSum = sum(unlocked.percentage)
  
  if oldUnlockedSum === 0:
    distribute equally among unlocked
  else:
    for each u in unlocked:
      u.percentage = (u.percentage / oldUnlockedSum) * remaining
      clamp(u.percentage, 0, 100)
```
